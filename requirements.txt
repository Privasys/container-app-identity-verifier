# Biometrics (face match + liveness), wired in verifier/biometrics.py. Imported
# lazily; required at runtime for the biometric step. OpenCV bundles the ONNX
# inference (YuNet detect + SFace recognise + MiniFASNet liveness) and decodes
# the DG2 portrait incl. JPEG2000 — no separate onnxruntime/Pillow needed.
# Models are fetched + checksum-pinned in the Dockerfile (not vendored).
#
# NOTE: cv2 is NOT pinned here. PaddleOCR (paddlex[ocr-core] below) requires the
# `opencv-contrib-python==4.10.0.84` DISTRIBUTION specifically (paddlex's dep
# check verifies the dist by name), so we let it provide the single cv2. It is a
# superset of the main modules face match uses (FaceDetectorYN/FaceRecognizerSF),
# so a separate opencv-python-headless would only conflict. The Dockerfile adds
# the GL/X runtime libs so this build imports on python:slim.
numpy>=1.26,<2                    # embedding maths (BSD); <2 for PaddleOCR compatibility

# Document OCR (GPG45 box 3 + the pre-NFC /read-mrz chip-key read): the data-page
# image is OCR'd IN THE ENCLAVE (the on-device camera OCR is unreliable on OCR-B).
# We drive PaddleOCR directly (verifier/doc_ocr.py) with the document pre-stages
# disabled and assemble the MRZ ourselves — OmniMRZ's bottom-50%/last-two-rows
# extractor read only fragments on real full-page photos (and its PyPI/source
# packaging ships no code anyway). PaddlePaddle (CPU) is installed from its own
# index in the Dockerfile; the det+rec models are baked in at build time so the
# enclave needs no network at runtime.
paddleocr>=3,<4                   # MRZ OCR engine (Apache-2.0)
# PaddleOCR 3.x is built on PaddleX, whose OCR pipeline needs extra runtime deps;
# without them PaddleOCR().predict raises DependencyError ("OCR requires
# additional dependencies"). `ocr-core` is the minimal inference set (no
# training/serving extras).
paddlex[ocr-core]>=3,<4          # PaddleOCR runtime deps (Apache-2.0)
